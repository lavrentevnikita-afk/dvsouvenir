# Roadmap проекта (B2B-only)

Этот roadmap фиксирует новый вектор: **Suvlavka работает только как B2B-платформа для заказов от наших магазинов**.

## Новые продуктовые границы

- Работаем только с заказами от магазинов-партнёров (роль `store`).
- Розничный сценарий исключаем из активной разработки.
- Блоки оплаты и доставки убираем из пользовательского потока заказа.
- Основной упор — максимально быстрый каталог и быстрый заказ для сотрудников магазина.
- Для админа фокус: управление каталогом, остатками, отгрузками и аналитикой.

---

## Целевое состояние (v1.1.0)

**Дата обновления:** 13 февраля 2026
**Цель:** упростить продукт до «операционного B2B-кабинета», убрать всё лишнее.

### 1) B2B-каталог и быстрый заказ (Store UX)

- [ ] Один «рабочий» каталог без маркетинговых блоков.
- [ ] Быстрый поиск по артикулу/названию с акцентом на скорость.
- [ ] Массовый ввод позиций (`артикул + количество`) и моментальное добавление в заказ.
- [ ] Шаблоны регулярных закупок («еженедельный», «как в прошлый раз»).
- [ ] Максимально короткий checkout: только нужные контактные данные/комментарий.

### 2) Админ-панель (операционка)

- [ ] Быстрое добавление и редактирование товара из одной панели.
- [ ] Управление остатками: приход, резерв, списание, инвентаризация.
- [ ] Управление отгрузками: создание, статусы, контроль SLA.
- [ ] Панель задач «в работе» для сборки и координации.

### 3) Аналитика для принятия решений

- [ ] Дашборд по заказам и отгрузкам (день/неделя/месяц).
- [ ] Отчёт по оборачиваемости и low-stock товарам.
- [ ] Топ SKU по количеству и выручке в B2B-сегменте.
- [ ] Воронка исполнения: создано → в работе → отгружено → закрыто.

---

## Что выносим из приоритета / удаляем

- Розничные пользовательские сценарии (витрина для частного клиента, retail-акции).
- UX-потоки, связанные с оплатой и доставкой в checkout.
- Функции, не влияющие на скорость заказа магазина и операционную эффективность.

---

## План миграции функционала

### Этап A (быстрый, 1 спринт)
- [ ] Зафиксировать B2B-only позиционирование в UI и документации.
- [ ] Скрыть или пометить deprecated retail-маршруты.
- [ ] Упростить форму заказа (убрать нецелевые поля).

### Этап B (2 спринта)
- [ ] Довести быстрый заказ и шаблоны до рабочего стандарта закупщика.
- [ ] Усилить админку по складу и отгрузкам.
- [ ] Добавить ключевые аналитические виджеты.

### Этап C (стабилизация)
- [ ] Удалить legacy-части retail после проверки метрик использования.
- [ ] Почистить API и DTO от неиспользуемых полей/эндпоинтов.
- [ ] Актуализировать тест-план под B2B-ядро.

---

## KPI после смены фокуса

- Время оформления типового заказа сотрудником магазина: **< 2 минут**.
- Добавление нового SKU админом: **< 1 минуты**.
- Доля заказов без ручных уточнений: **> 95%**.
- Полнота данных по остаткам и отгрузкам на дашборде: **100% ключевых SKU**.
